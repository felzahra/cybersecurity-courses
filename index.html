📡 LESSON 4.10

Monitoring Network Traffic

Deep packet inspection · Flow analysis · Anomaly detection

📖 Why monitor network traffic? Continuous surveillance helps identify malicious activities, performance bottlenecks, and policy violations. In this module, you’ll learn about flow data (NetFlow/IPFIX), packet capture (PCAP), and how to spot command-and-control patterns.

✅ Key tools: tcpdump, Wireshark, Zeek (formerly Bro), ntopng.

✅ Common threats detected: port scanning, DDoS patterns, data exfiltration, ARP spoofing.

✅ Hands-on concept: Capture live traffic and filter for suspicious DNS queries or unusual outbound connections.

🔎 After this video, review the quiz (4.11) and then proceed to the SIEM framework. Monitoring is the first pillar of any Security Operations Center (SOC).

🛡️ LESSON 4.14–4.15

SIEM Framework

Security Information and Event Management · Log aggregation · Correlation

📌 SIEM (Security Information and Event Management) combines SIM (Security Information Management) and SEM (Security Event Management). Learn how SIEM platforms collect logs from firewalls, servers, endpoints, and network devices to provide real-time analysis.

⚙️ Core components: Data collection (agents/forwarders), correlation engine, alerting dashboard, and long-term storage.

📈 Popular SIEM solutions: Splunk ES, IBM QRadar, Microsoft Sentinel, ELK Stack (Elastic Security), Sumo Logic.

📋 Use case: Correlate failed login attempts + unusual process creation to detect brute-force attacks.

This video covers SIEM architecture, log normalization, and how to build detection rules. Exercises 4.16 and 4.17 will walk you through a simulated SIEM investigation.

📡 EXERCISE 4.12–4.13

Traffic Analysis & Packet Capture

Hands-on with Wireshark · Analyzing PCAPs

📂 Exercise scenario: You are given a suspicious PCAP file containing traffic from an infected workstation. Identify the malicious indicators: beaconing domains, unusual ports, and potential data leak.

🛠️ Analysis steps:

1️⃣ Filter HTTP requests: http.request → locate unusual user-agents.
2️⃣ Check DNS queries for algorithmically generated domains (DGA).
3️⃣ Use Statistics → Protocol Hierarchy to detect unexpected protocols.
4️⃣ Follow TCP streams to reassemble exfiltrated data.

💡 Pro tip: Combine tshark and grep for command-line analysis. In a SOC, automated packet analysis scripts reduce mean time to detect (MTTD).

🔎 EXERCISE 4.16–4.17

Threat Hunting with SIEM

Query languages · Detection rules · Incident response

🎯 Real-world SIEM investigation: This exercise replicates a lateral movement attack. Using SIEM search (SPL/KQL), you will correlate Windows Event IDs 4624 (logon) and 5140 (network share access) to discover abnormal behavior.

📌 Detection logic example:
index=windows sourcetype=WinEventLog:Security EventCode=4624 AccountName!="SYSTEM"
→ filter out system accounts and map unusual source IPs.

✅ Outcome: Build a detection rule for pass-the-hash attempts and schedule automated alerts.

🏆 After the exercise solution review (4.17), you'll understand how to tune SIEM rules to reduce false positives.

🔐 Proactive threat hunting reduces breach dwell time. Combine SIEM with EDR for next-gen defense.
